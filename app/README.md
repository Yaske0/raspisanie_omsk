# Расписание смены — Омский аэропорт

Программа для одной кнопки: определяет вашу текущую/ближайшую смену,
скачивает онлайн-табло Омского аэропорта (прилёт и вылет, с учётом задержек),
формирует документ Word по вашему образцу и отправляет его на печать.

- Шрифт **Verdana**, заголовки 28, текст 11, всё на одной странице.
- Учитываются задержки: рейс, задержанный «в вашу смену», попадает в список;
  рейс, уехавший за конец смены, — убирается.
- Новое время прилёта/вылета подставляется автоматически; если оно ещё
  неизвестно — пишется **ЗАДЕРЖАН**.
- График смен: цикл 4 дня (день → ночь → выходной → выходной), якорь 25.06.2026.

## Как получить готовый .exe (без установки чего-либо на ваш ПК)

Сборка происходит в облаке GitHub. На вашем компьютере ничего ставить не нужно —
только браузер. Один раз настраиваем, дальше просто пользуемся файлом.

1. Заведите бесплатный аккаунт на https://github.com (если ещё нет).
2. Создайте новый репозиторий: кнопка **New** → имя любое (например `raspisanie`)
   → можно **Private** → **Create repository**.
3. Загрузите файлы проекта: на странице репозитория **Add file → Upload files**,
   перетащите туда **всё содержимое** этой папки (включая папку `.github`)
   → **Commit changes**.
   > Если папка `.github` не загружается перетаскиванием, создайте файл вручную:
   > **Add file → Create new file**, в поле имени введите
   > `.github/workflows/build.yml` и вставьте его содержимое.
4. Откройте вкладку **Actions**. Сборка запустится сама (или нажмите
   **Run workflow**). Идёт 3–5 минут.
5. Когда напротив запуска появится зелёная галочка — откройте его и внизу
   в разделе **Artifacts** скачайте **РасписаниеСмены-windows** (это zip).
6. Распакуйте zip. Внутри — `РасписаниеСмены.exe` и `config.json`.
   Положите их вместе в любую папку. Готово.

Дальше каждый раз: **двойной клик по `РасписаниеСмены.exe` → кнопка
«Создать и напечатать»**. Документ уйдёт на принтер по умолчанию.

## Настройки — файл `config.json` (необязательно)

Лежит рядом с exe, открывается Блокнотом. Пересобирать ничего не нужно.

```json
{
    "anchor_day_shift": "2026-06-25",
    "printer_name": "",
    "auto_print": true,
    "output_dir": ""
}
```

- **anchor_day_shift** — любая дата, когда у вас точно была **дневная** смена.
  От неё считается цикл. Поменяйте, если график «сдвинулся» (например, после
  подмены). Формат `ГГГГ-ММ-ДД`.
- **printer_name** — точное имя принтера (как в «Устройствах и принтерах»).
  Пусто = принтер по умолчанию.
- **auto_print** — `true` печатать сразу, `false` только создавать файл.
- **output_dir** — куда складывать документы. Пусто = папка «Расписания»
  рядом с exe.

## Что внутри (для понимания / доработки)

| Файл | Назначение |
|------|------------|
| `aeroomsk/shifts.py` | расчёт смены по 4-дневному циклу |
| `aeroomsk/tablo.py` | загрузка и разбор табло (`type=ARR/DEP`, `day=PREV/THIS/NEXT`) |
| `aeroomsk/builder.py` | отбор рейсов под смену и правила задержек |
| `aeroomsk/docxgen.py` | сборка Word по образцу (Verdana 28/11, рамки, 1 страница) |
| `aeroomsk/printing.py` | тихая печать на Windows (через Word, затем системно) |
| `aeroomsk/core.py` | связывает всё вместе |
| `aeroomsk/app.py` | окно с кнопкой |

Источник данных — онлайн-табло: `https://www.aeroomsk.ru/?type=ARR&day=THIS&AjaxTablo=ajax`
(аналогично `DEP`, и `day=NEXT/PREV`).

## Если что-то не так

- **Окно открылось, но при создании — ошибка сети.** Проверьте, что с рабочего
  ПК открывается сайт aeroomsk.ru. Возможно, нужен доступ/прокси.
- **Печать не пошла, файл создан.** Откройте папку «Расписания» (кнопка
  «Открыть папку»), напечатайте вручную; либо укажите `printer_name` в config.
- **Не та смена.** Поправьте `anchor_day_shift` в `config.json`.
- **Сайт поменял разметку табло.** Разбор устойчив к мелким изменениям, но при
  крупной переделке сайта может потребоваться правка `tablo.py`.
